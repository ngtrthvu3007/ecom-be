Simple API for Ecommerce

This repo is a simple api server for a ecommerce website. It was created to provide data in a free, simple and fast way for researching and understanding libraries or frameworks on the Client side such as React, Vue, Svelte

Repo-url: https://github.com/ngtrthvu3007/ecom-be

Domain: https://vu-ecom-api.onrender.com/

routes:

users: https://vu-ecom-api.onrender.com/users/{path}

path:

  • my-profile GET
  • register POST >> params: { *required params [name: string, password: string, confirm_password: string, email: string, phone: string, address: string], *optional [isAdmin: boolean] }
  • login POST >> params: { email: string, password: string }
  • update-my-profile PUT >> [Authorization]* update only params: { name: string, address: string }
  • change-password PUT >> [Authorization]* params: { new_password: string, password: string, confirm_password: string}
    • products: https://vu-ecom-api.onrender.com/products{path}

    path:

  • /:sku GET >> get detail of a product
  • /?{query} GET >> get products with query is sort/filter/limit/page >> example: {{domain}}/products?sort=price&sort=desc&filter=sku&filter=ip11
  • create POST >> [Authorization]* params: { *required params [sku: string, name: string, image: string, type: string, price: int, count_in_stock: number], *optional [rating: number int(1-5),description: string ] }
  • delete/:sku DELETE >> [Authorization]*
  • update/:sku PUT >> [Authorization]* update only params: {*optional [ name: string, price: int, count_in_stock: number , rating: number int(1-5) ]}
    • orders: https://vu-ecom-api.onrender.com/orders{path}

    path:

  • /:idGET >> [Authorization]* >> get detail of a order
  • ?{query} GET >> [Authorization]* status['pending', 'done', cancel'] >>get orders with query is status >> example: {{domain}}/orders?status=pending
  • create POST >> [Authorization]* params: { *required params [address: string, phone: string, paymentMethod: string/default: "cash", products: Array[ Object{ product_id: string, product_amount: int }] ]} create a order
  • /update/:id PUT >> [Authorization]* update only params: {*optional [ address: string, paymentStatus: boolean, deliveryStatus: string: ['pending', 'shipping', 'done'] , orderStatus: string : ['pending', 'shipping', 'done'] ]}
